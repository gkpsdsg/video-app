- 成功构建 50.8MB release APK

### HTTP 明文拦截 — `3f59f83`

**问题：** Android 9+ 默认拦截 HTTP 明文请求，登录无响应，抓包工具无数据。

**修复：**
- AndroidManifest.xml 添加 `android:usesCleartextTraffic="true"`
- 创建 `network_security_config.xml` 允许明文流量

### 网络连通调试 — `90ebbc0`

**问题：** 手机连不上服务器，逐层排查网络链路：

```
手机 (WiFi) → 宿主机 → 虚拟机(后端:3000 + MinIO:9000)
```

| 排查步骤 | 状态 | 说明 |
|----------|------|------|
| 虚拟机内 `curl localhost:3000` | ✅ | 后端正常运行 |
| 宿主机 `curl 192.168.161.128:3000` | ✅ 修复 | VMnet8 静态 IP 丢失，`netsh` 重设 |
| 宿主机 `curl localhost:3000` | ✅ 修复 | `netsh portproxy` 重启后丢失，重新添加 |
| 手机浏览器 `http://IP:3000` | ✅ 修复 | 防火墙规则重启后丢失，重新添加 |
| APK 登录 | ✅ 修复 | release manifest 缺少 `INTERNET` 权限 |
| 视频加载 | ✅ 修复 | MinIO 预签名 URL 用 `localhost`，改外部 IP + 端口转发 |

**最终网络架构：**

```
手机 (WiFi: 10.198.238.x)
  │
  ├─ API 请求 → 10.198.238.77:3000 ──portproxy──▶ VM 192.168.161.128:3000 (NestJS)
  │
  └─ 视频文件 → 10.198.238.77:9000 ──portproxy──▶ VM 192.168.161.128:9000 (MinIO)
```

**宿主机持久化规则（需管理员 CMD）：**
```cmd
netsh interface portproxy add v4tov4 listenport=3000 listenaddress=0.0.0.0 connectport=3000 connectaddress=192.168.161.128
netsh interface portproxy add v4tov4 listenport=9000 listenaddress=0.0.0.0 connectport=9000 connectaddress=192.168.161.128
netsh advfirewall firewall add rule name="Port 3000" dir=in action=allow protocol=TCP localport=3000
netsh advfirewall firewall add rule name="Port 9000" dir=in action=allow protocol=TCP localport=9000
```

**其他修复：**
- 登录/注册添加 DioException 错误捕获和中文错误提示 SnackBar
- 后端 `main.ts` 添加 MinIO 配置日志，便于诊断

## 数据库表结构

| 表 | 主要字段 |
|----|---------|
| `users` | id, username, password, nickname, avatar, role, createdAt |
| `videos` | id, title, minioObjectName, coverObjectName, status(UPLOADING/TRANSCODING/READY/FAILED), visibility(public/private), playCount, likeCount, commentCount, authorId |
| `follows` | id, followerId, followingId, createdAt（唯一约束） |
| `comments` | id, content, userId, videoId, parentId, createdAt |
| `likes` | id, userId, videoId, createdAt（唯一约束） |
| `bookmarks` | id, userId, videoId, createdAt（唯一约束） |
| `notifications` | id, recipientId, actorId, type(like/comment/follow), targetId, isRead, createdAt |
| `conversations` | id, participant1Id, participant2Id, lastMessageAt |
| `messages` | id, conversationId, senderId, content, readAt, createdAt |
| `reviews` | id, videoId, reviewerId, status(pending/approved/rejected), reviewType(auto/manual), reason |

## 已知小问题

1. IP 地址硬编码在 `main.dart` 和 `.env` 中，切换网络需手动修改 + 重编译
2. 宿主机重启后 `portproxy` 和防火墙规则会丢失，需重新添加
3. 部分页面功能尚未完全对接（如手机号注册、搜索历史等占位功能）
4. 视频播放不支持 Range 请求，长视频拖动可能不够流畅
